Hexo使用手册

Hexo优势

  • 中文杜区支持完善：拥有大量中文教程和主题资源，适合中文用户。
  • 丰富的主题生态：支持300+主题，可快速实现个性化页面设计。
  • 基于Node.js生态系统：插件和模块丰富，安装配置简单。
  • 插件系统灵活：可扩展性强，满足SEO、图片处理、搜索等多种需求。
  • 低学习曲线：上手简单，适合初学者快速构建博客。 Markdown渲染性能优秀：书写体验流畅，渲染效果出色。

Hexo常用命令表

命令 简写 说明
hexo init [folder] - 初始化一个新的Hexo项目
hexo new "文章标题" hexo n 创建一篇新文章
hexo new draft "文章标题" - 创建一篇草稿文章
hexo new page "页面名称" - 创建一个新页面
hexo generate hexo g 生成静态文件(构建网站)
hexo server hexo s 启动本地服务器预览网站
hexo deploy hexo d 部署网站到远程仓库
hexo clean - 清楚缓存文件
hexo list - 列出站点中的内容
hexo version - 显示Hexo版本信息

github部署

1. 建立github源码私有库及public网页库

分别设置为==public==库和==pravite==库

2.配置Tokens(classic)

  • 在Settings>Developer Settings>Personal access tokens>Tokens(classic)中建立token，Token至少带repo, workflow权限。
  • 进入源码私有库中，Settings>Secrets and variables>Actions中将token添加（token要具有workflow工作流权限）

3.实现自动化

  1. 将Hexo源码根目录推送到私有库中

  1. 修改_config.yml

    在_config.yml中修改deploy字段。指示Hexo在deploy(生成静态页面)时的推送地址。（此处的推送地址为public库）

  2. 配置Github Action工作流

    在.github文件夹下新增workflows文件夹，然后新增deploy.yml文件，内容如下。(node-version要与本地的node保持一致)。修改node版本，推送库地址，token。(yml脚本意为使用ubuntu-latest作为基础环境，安装依赖环境，随后hexo generate生成博客网站静态文件夹，最后把这个文件夹推送到公开仓库的main分支)

    查看私有库Action中的工作流是否正常运行，若正常运行则核验公有库中推送到的代码
